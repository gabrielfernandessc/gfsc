Como Desinstalar e Reinstalar um Driver de Rede no Gerenciador de Dispositivos (Dell ou Lenovo)

Objetivo

Este guia descreve como desinstalar e reinstalar um driver de rede no Gerenciador de Dispositivos em computadores Dell ou Lenovo, útil para resolver problemas de conectividade ou atualizar drivers.

Pré-requisitos

  • Acesso administrativo ao computador.
  • Sistema operacional Windows (Windows 7, 8, 10 ou 11).
  • Conexão alternativa (ex.: cabo USB ou outra rede) caso a conectividade seja perdida.

Passo a Passo

Parte 1: Desinstalação do Driver

  1. Abrir o Gerenciador de Dispositivos
    Pressione Windows + R, digite devmgmt.msc e pressione Enter.
  2. Localizar o Adaptador de Rede
    Expanda a seção Adaptadores de Rede e identifique o dispositivo (ex.: "Realtek PCIe GBE Family Controller").
  3. Desinstalar o Driver
    Clique com o botão direito no adaptador, selecione Desinstalar dispositivo, marque Excluir o software de driver (se disponível) e clique em Desinstalar.
  4. Aguardar a Conclusão
    O driver será removido, e o dispositivo pode desaparecer da lista ou aparecer como "Dispositivo desconhecido".

Parte 2: Reinstalação do Driver (Dell ou Lenovo)

  1. Verificar Reinstalação Automática
    Reinicie o computador. O Windows pode reinstalar um driver padrão. Teste a conexão.
  2. Baixar o Driver Específico
    • Para Dell: Acesse www.dell.com/support, insira o Service Tag, localize "Rede" e baixe o driver.
    • Para Lenovo: Visite support.lenovo.com, use o número de série, filtre por "Rede" e baixe o driver.
  3. Instalar o Driver
    Execute o arquivo .exe baixado ou, para .zip, use Atualizar driver no Gerenciador de Dispositivos apontando para a pasta extraída.
  4. Reiniciar o Computador
    Reinicie e teste a conectividade.

Notas Adicionais

  • Baixe drivers compatíveis com sua versão do Windows (32 ou 64 bits).
  • Se a conexão for perdida, use um USB ou outra máquina para transferir o driver.

Solução de Problemas

  • Driver não aparece: Verifique o BIOS/UEFI.
  • Erro na desinstalação: Execute como administrador.
  • Driver incorreto: Confirme o modelo do adaptador nas propriedades.
